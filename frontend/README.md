# Frontend — Socratic AI Tutor (React + Vite)

The React SPA. Talks to **Supabase** directly for auth / storage / table
reads, and to the **Node backend** for the AI tutor (SSE) and simulator
endpoints. Designed to deploy to **Vercel**.

## Tech

- React 18 + TypeScript + Vite 5
- Tailwind v3 + shadcn/ui
- React Router 6 (`BrowserRouter`)
- `@supabase/supabase-js` (auth, storage, table reads)
- KaTeX + react-markdown for math rendering

## Local development

```bash
# 1. Install
npm install

# 2. Configure
cp .env.example .env
# Fill VITE_SUPABASE_URL, VITE_SUPABASE_PUBLISHABLE_KEY, VITE_API_URL

# 3. (in another terminal) start the backend on :8787
#    cd ../backend && npm run dev

# 4. Run the dev server
npm run dev          # → http://localhost:5173
```

## Build

```bash
npm run build        # → dist/
npm run preview      # serve the production build locally
```

## Deploy: Vercel

1. <https://vercel.com> → **Add New → Project** → import your repo.
2. Set **Root Directory** to `frontend`.
3. Framework preset: **Vite** (Vercel auto-detects). Build command
   `npm run build`, output `dist`.
4. Environment Variables:

   | Name                            | Value                                      |
   |---------------------------------|--------------------------------------------|
   | `VITE_SUPABASE_URL`             | `https://<your-ref>.supabase.co`           |
   | `VITE_SUPABASE_PUBLISHABLE_KEY` | your Supabase anon key                     |
   | `VITE_SUPABASE_PROJECT_ID`      | your Supabase project ref (optional)       |
   | `VITE_API_URL`                  | `https://<your-backend>.onrender.com`      |

5. Deploy. The included `vercel.json` handles SPA fallback so deep links
   like `/session/<id>` work after a page refresh.

After the first deploy, also add the Vercel domain to:

- The backend's `ALLOWED_ORIGINS` env var (Render → Environment).
- Your Supabase project's **Authentication → URL Configuration →
  Redirect URLs**.

## Notes

- `src/integrations/supabase/client.ts` and `src/integrations/supabase/types.ts`
  were originally auto-generated by Lovable Cloud. After switching to your
  own Supabase project, you can regenerate the types:

  ```bash
  npx supabase gen types typescript \
    --project-id <your-project-ref> \
    > src/integrations/supabase/types.ts
  ```

  The `client.ts` file already reads from Vite env vars, so no edits are
  needed there.
